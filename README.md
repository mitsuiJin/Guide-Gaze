## 프로젝트 개요

본 프로젝트는 아이트래킹(Eye-Tracking) 기술을 단순 보조 입력 수단이 아닌, 주도적이고 풍부한 입력 방식으로 확장하는 것을 목표로 한다. 곡선 기반 시각 경로(Color Lane)와 시선 속도 유사성 분석을 결합한 입력 시스템을 설계하여 기존 시선 입력 방식의 한계(Midas Touch 문제 등)를 극복하고, 운동 장애인 등 손을 자유롭게 사용할 수 없는 사용자가 시선만으로 복합 단축키 명령을 정확하고 빠르게 수행할 수 있도록 하였다.

## 기존 연구의 한계

1. **정확도 개선 중심의 접근**: 기존 시선 입력 연구는 커서 대체·정확도 향상에 초점을 두었으며, 실제 작업 환경의 복합 단축키 입력까지 확장하는 데는 한계가 있었다.
2. **입력 다양성 부족**: 가상 키보드/커서 조작 등 단일 입력 방식 위주이며, 다중 명령 조합을 빠르게 수행하기 어렵다.
3. **정확한 포인팅 요구**: 시선 오차·노이즈 발생 시 오입력 가능성이 높으며, 실제 환경에서 안정적 사용이 어렵다.

## 시스템 구성

시스템은 다음 세 가지 모듈로 구성된다.

### 1. Gaze Input 처리 모듈

- Tobii Eye Tracker로 시선 좌표 실시간 획득
- 궤적 추출, 속도 계산 등 전처리 수행

### 2. Color Lane UI 모듈

- 단축키 조합을 곡선(Color Lane)으로 시각화
- UI 오브젝트가 Ctrl → 각 키 방향으로 이동

### . 입력 판별 알고리즘

- 프레셰 거리 기반 경로 유사도 + 속도 패턴 유사도 결합
- 가장 유사한 곡선에 대응된 명령을 실행

# 알고리즘

## 1. 프레셰 거리 기반 경로 유사도

### 개념 및 역할

- 시선 궤적과 Color Lane 곡선의 전체적인 형태 유사도 평가
- 시작과 끝, 궤적 순서와 곡률을 모두 반영
- 단순 점 간 거리 비교 방식보다 사용자의 의도를 더 정확히 반영

### 판별 과정

- 시선 좌표 시퀀스 vs 레인 좌표 시퀀스 비교
- 값이 작을수록 높은 유사도
- 형태 유사도가 높은 곡선이 다수일 때 속도 기준 추가 적용

## 2. 속도 유사도

### 도입 배경

- 곡선 형태가 유사한 경우 오분류 가능성 존재

### 방법

- 시선 속도 vs 곡선 위 객체 속도 비교
- 차이가 작을수록 높은 유사도

## 3. Color Lane 설계 전략
### 레인 수 제한 (4개)

- 인지 부하 최소화 및 시각적 간섭 방지
- 시작점(키보드 Ctrl) 기준 상·하·좌·우 4방향으로 경로 분리

### 곡률 차별화

- Left/Top: 위로 볼록
- Right/Bottom: 아래로 볼록
- 곡선 간 교차를 구조적으로 방지하여 입력 모호성 최소화

## 3. 성능 분석

| 지표 | 결과 |
| --- | --- |
| 입력 성공률 | 약 95% |
| 입력 대상 | 4개(Color Lane) |
| 총 시도 횟수 | 120회 |
| 오입력 감소 요인 | 곡선 형태 분리 + 속도 유사도 적용 |
